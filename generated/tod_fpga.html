纳秒累加器 — 核心时间推进

SystemVerilog // 每时钟周期累加一个"ns_increment" // CLK=125MHz → ns_per_tick = 8ns // 小数部分用 16-bit 定点扩展，等效 ~0.015ps 分辨率 localparam logic [45:0] NS_INCREMENT = 46'd8 << 16; // 8ns with 16 fractional bits logic [45:0] ns_accumulator; always_ff @(posedge clk or negedge rst_n) begin if (!rst_n) begin ns_accumulator <= '0; tod_seconds_o <= '0; tod_ns_o <= '0; end else begin ns_accumulator <= ns_accumulator + NS_INCREMENT; // ns_accumulator[45:16] = integer nanoseconds if (ns_accumulator[45:16] >= 1_000_000_000) begin ns_accumulator[45:16] <= ns_accumulator[45:16] - 1_000_000_000; tod_seconds_o <= tod_seconds_o + 1; end tod_ns_o <= ns_accumulator[45:16]; end end assign tod_ns_frac_o = ns_accumulator[15:0];

频率调整 (PPB 补偿)

SystemVerilog // 频率调整通过改变每周期增量实现 // 1 ppb → 每 10^9 ns 调整 1ns // 使用 32-bit 定点相位累加器实现分数调整 logic [31:0] phase_acc; always_ff @(posedge clk or negedge rst_n) begin if (!rst_n) begin phase_acc <= '0; end else begin phase_acc <= phase_acc + freq_adj_i; // 相位累加器溢出 → 执行 1ns 额外调整 if (phase_acc[31]) begin ns_accumulator <= ns_accumulator + (1 << 16); phase_acc[31] <= 1'b0; end end end

FPGA 实现要点

跨时钟域 (CDC)

ToD 值需在多个时钟域间传递。推荐使用 Gray 码编码秒+纳秒字段，或采用握手/FIFO 方式安全传递多比特时间戳。

亚纳秒精度

通过 16-bit 定点小数扩展纳秒累加器，在 125MHz 时钟下可达到 ~0.015ps 的表示精度，满足 IEEE 1588 PTP 的时间戳分辨率需求。

频率补偿

使用 32-bit 相位累加器实现 PPB (Parts Per Billion) 级频率调整。相位累加器每次累加 freq_adj 值，溢出时产生 1ns 校正。

步进调整

支持有符号 64-bit 步进调整量，用于 PTP 同步算法中的大偏差校正。步进操作需在单周期内原子完成，避免中间态被读出。

时间戳快照

PTP 时间戳请求信号触发后，当前 ToD 值被锁存到专用寄存器。快照操作需与累加器更新在同一周期完成，保证时间戳一致性。

BRAM / 寄存器

ToD 核心逻辑仅需少量 FF (~150) 和一个 DSP48 (用于乘法累加)。若需存储历史时间戳，可使用 BRAM 实现 FIFO。
